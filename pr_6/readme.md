### 2. Мета роботи

Метою практичної роботи є проектування та реалізація захищеного веб-сервісу для управління системою теплових насосів житлового комплексу. Основна увага приділяється впровадженню багаторівневої системи автентифікації (за допомогою Passport.js) та авторизації з контролем доступу на основі ролей (RBAC) та атрибутів (ABAC), а також захисту API від поширених веб-вразливостей (XSS, CSRF, Brute-force).

### 3. Теоретична частина

**Опис об'єкта автоматизації:**
Об'єктом автоматизації є IoT-система теплових насосів багатоквартирного будинку. Система обслуговує три категорії користувачів: жителів (керування мікрокліматом власної квартири), техніків (обслуговування обладнання) та менеджерів комплексу (моніторинг загальної енергоефективності).

**Перелік загроз безпеки для даного типу системи:**

1. Несанкціонований доступ до панелі керування насосами.
2. Перехоплення сесій користувачів (Session Hijacking).
3. Міжсайтовий скриптінг (XSS) через невідфільтровані вхідні дані.
4. Підробка міжсайтових запитів (CSRF) для прихованої зміни температурних режимів.
5. Атаки типу Brute-force на форму авторизації.

**Обґрунтування вибраних методів захисту:**

* **Passport.js (Local Strategy):** Забезпечує надійний механізм локальної перевірки облікових даних із застосуванням `bcrypt` для хешування паролів.
* **Helmet.js & HttpOnly Cookies:** Блокують виконання шкідливих скриптів та закривають доступ до сесійних даних на стороні клієнта (захист від XSS).
* **SameSite: Strict:** Ізолює файли cookie від міждоменних запитів, ефективно нейтралізуючи вектори CSRF-атак без необхідності генерації додаткових токенів.
* **Express-rate-limit:** Мінімізує ризики Brute-force та DDoS шляхом обмеження частоти запитів.
* **Zxcvbn & Express-validator:** Запобігають використанню компрометованих/слабких паролів та санітизують вхідні дані (захист від ін'єкцій).

### 4. Опис реалізації

**4.1. Архітектура додатку**
Додаток побудовано за трирівневою (Layered) архітектурою:

1. **Шар маршрутизації (Routes):** Визначає endpoints та підключає middleware.
2. **Шар контролерів (Controllers):** Обробляє HTTP-запити та формує відповіді.
3. **Шар сервісів (Services):** Містить ізольовану бізнес-логіку (перевірка криптостійкості паролів, генерація токенів верифікації).

**5.2. Тестування безпеки**

* **Спроба XSS атаки:** Не працює, оскільки дані проходять через `express-validator` (метод `.escape()`), а сесійні кукі захищені прапорцем `httpOnly`, що унеможливлює крадіжку сесії через JavaScript.
* **Спроба CSRF атаки:** Блокується на рівні браузера завдяки налаштуванню cookie `SameSite: strict`.
* **Спроба доступу без автентифікації:** Запити до `/api/*` без сесійної кукі повертають помилку 401 Unauthorized завдяки middleware `isAuthenticated`.
* **Спроба доступу з недостатніми правами:** Запит від користувача з роллю `resident` на endpoint `/api/system/efficiency` повертає помилку 403 Forbidden (відпрацьовує `checkRole`).
* **Тестування rate limiting:** Після 5 невдалих спроб входу з одного IP-адреси, сервер повертає помилку `429 Too Many Requests`.

### 6. Результати роботи

* Розроблено захищений REST API для управління IoT-системою.
* Впроваджено багатофакторну перевірку при реєстрації (складність пароля, підтвердження електронної пошти).
* Реалізовано надійний механізм сесій із захистом від Session Fixation (регенерація ID після логіну) та безпечною функцією "Remember Me" (маніпуляція часом життя `maxAge`).
* Створено систему розмежування прав доступу, яка враховує не лише роль (RBAC), а й приналежність об'єкта керування конкретному користувачу (квартира жителя - ABAC).

### 7. Висновки

Під час виконання практичної роботи було реалізовано комплексну систему безпеки веб-додатку на базі Node.js. Здобуто практичні навички роботи з бібліотекою Passport.js для організації життєвого циклу сесій, використання middleware для санітизації вхідних даних та реалізації механізмів захисту від базових веб-вразливостей (OWASP Top 10). Запропонована архітектура дозволяє легко масштабувати систему та додавати нові IoT-пристрої або ролі користувачів без зниження рівня безпеки.
